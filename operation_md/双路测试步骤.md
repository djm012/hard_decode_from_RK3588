# 双路测试步骤

## 1. 目标

把两个本地视频分别走完整链路：

- `EasyDarwin.mp4`
- `sec_branch.mp4`

链路如下：

```text
EasyDarwin.mp4 -> RTSP Simulator -> EasyGBD 实例1 -> WVP -> ZLMediaKit -> RK3588 硬解码
sec_branch.mp4 -> RTSP Simulator -> EasyGBD 实例2 -> WVP -> ZLMediaKit -> RK3588 硬解码
```

这里的“双路”不是一个视频开两个窗口，而是两个独立视频源同时注册、同时点播、同时输出 RTSP、同时被 RK3588 拉流硬解码。

## 2. 固定参数

RK3588 当前测试按下面这组参数：

- WVP Web：`18080`
- SIP：`5060`
- SIP 平台 ID：`34020000002000000001`
- SIP 密码：`12345678`
- ZLM IP：`172.10.50.34`
- ZLM HTTP：`9092`
- ZLM RTSP：`554`
- ZLM secret：`TWSYFgYJOQWB4ftgeYut8DW4wbs7pQnj`

WVP 页面：

```text
http://172.10.50.34:18080
```

## 3. Windows 上先生成两路 RTSP

### 3.1 启动 RTSP Simulator

双击运行：

```text
EasyRTSPServer_Demo.exe
```

### 3.2 把两个视频放到 exe 同目录

- `EasyDarwin.mp4`
- `sec_branch.mp4`

### 3.3 两路 RTSP 地址

```text
rtsp://127.0.0.1:554/EasyDarwin.mp4
rtsp://127.0.0.1:554/sec_branch.mp4
```

### 3.4 先验证 RTSP 能不能播

```bash
ffplay rtsp://127.0.0.1:554/EasyDarwin.mp4
ffplay rtsp://127.0.0.1:554/sec_branch.mp4
```

## 4. 启动两个 EasyGBD 实例

单设备版 EasyGBD 建议复制两份目录，分别启动：

- `EasyGBD_1`
- `EasyGBD_2`

## 7. WVP 上要做什么

打开：

```text
http://172.10.50.34:18080
```

确认：

- 两个设备都在线
- 两个通道都在线
- 两个通道都分别点击“播放”

注意：双路里最容易漏掉的就是只点了一路播放。只点一路时，另一条 RTSP 往往不会持续可用。

## 8. 双路 RTSP 输出地址

你现在已经得到这两路：

```text
rtsp://172.10.50.34:554/rtp/34020000001110000001_34020000001310000001?originTypeStr=rtp_push
rtsp://172.10.50.34:554/rtp/34020000001110000002_34020000001310000002?originTypeStr=rtp_push
```

## 9. 先验证两路 RTSP 都能拉到

```bash
ffplay "rtsp://172.10.50.34:554/rtp/34020000001110000001_34020000001310000001?originTypeStr=rtp_push"
ffplay "rtsp://172.10.50.34:554/rtp/34020000001110000002_34020000001310000002?originTypeStr=rtp_push"
```

如果其中一路 404，先回到 WVP 重新点这一路播放，再复制最新 RTSP 地址。

## 10. RK3588 上做双路硬解码

### 方式 1：两个独立进程

这是最稳的验证方式。

```bash
python3 rkmpp_video_player_optimized.py "rtsp://172.10.50.34:554/rtp/34020000001110000001_34020000001310000001?originTypeStr=rtp_push" > stream1.log 2>&1 &
python3 rkmpp_video_player_optimized.py "rtsp://172.10.50.34:554/rtp/34020000001110000002_34020000001310000002?originTypeStr=rtp_push" > stream2.log 2>&1 &
```

查看日志：

```bash
tail -f stream1.log
tail -f stream2.log
```

### 方式 2：一个进程传两个 RTSP

如果你板子上的脚本已经支持多参数：

```bash
python3 rkmpp_video_player_optimized.py \
  "rtsp://172.10.50.34:554/rtp/34020000001110000001_34020000001310000001?originTypeStr=rtp_push" \
  "rtsp://172.10.50.34:554/rtp/34020000001110000002_34020000001310000002?originTypeStr=rtp_push"
```

## 11. 双路成功标准

满足下面这些，就说明双路基本跑通：

1. RTSP Simulator 能同时提供两路 RTSP
2. 两个 EasyGBD 都能持续运行
3. WVP 上看到 2 台设备 / 2 个通道在线
4. 两个通道分别点播后，都能复制出 RTSP
5. 两路 `ffplay` 都能正常打开
6. RK3588 上两路日志都持续输出 `decode_total_avg` / `process_fps`

## 12. 最常见的失败点

1. 两个 EasyGBD 用了相同的本地端口
2. 两个 EasyGBD 用了相同的设备 ID
3. 两个 EasyGBD 用了相同的通道 ID
4. 只点播了一路，另一条流没有保持活跃
5. RTSP Simulator 里的两个视频没有同时可读
6. 复制 RTSP 地址太早，WVP 还没把那一路真正拉起来
